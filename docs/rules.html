アップデート履歴管理ルール

  • アップデート履歴（data/updates.json）は、機能追加・修正・改善などのたびに必ず更新する。
  • 各履歴エントリには、以下の項目を必ず含める：
    • version: バージョン番号（例: "1.0.1"）
    • date: 年月日＋更新時刻（例: "2024-05-01 14:30"）
    • title: アップデートタイトル
    • changes: 主な変更点（配列）
    • techPoints: 技術的ポイント（配列、省略可）
    • summary: 概要・要約
  • dateは「YYYY-MM-DD HH:mm」形式で記載すること。
  • すべての履歴は配列として管理し、最新が先頭に来るようにする。
  • ルールや運用方針の変更も履歴に記載すること。

フォーマット例

{ "version": "1.0.1", "date": "2024-05-01 14:30", "title": "新機能追加", "changes": ["〇〇機能の追加", "バグ修正"], "techPoints": ["API連携改善"], "summary": "新機能追加と安定性向上のアップデートです。" }

サイト管理ルール

  • bocchy-project-siteはDiscord Bot「ボッチー」の紹介・説明を目的とした公開Webサイトプロジェクトである。
  • ボッチー本体（C:/Users/seamo/Documents/cursor/bocchy）とは完全に分離して管理する。
  • サイトの構成は以下の通り：
    • docs/: 静的サイト（HTML, CSS, JS）で実際に公開される内容
    • data/: サイトに反映するデータ（updates.jsonなど）
    • rules/: 運用・設計・デザイン等のメタ的なルール・マニフェスト
  • HTMLコンテンツはグラフィックレコーディング風のインフォグラフィック手法を用いて、視覚的に分かりやすく情報を提示する。
  • 色彩設計は「山の墨」「海霧の灰」「朝露の藍」「焙煎の琥珀」「柚葉の黄」の5色を基調とする。
  • 手書き風フォント（Yomogi, Zen Kurenaido, Kaisei Decol）を使用し親しみやすいデザインを維持する。
  • ボッチーの主要機能（AI応答・検索連携・記憶管理・空気読み・頻繁な進化）を分かりやすく情報を提示する。
  • サイト内の情報は定期的に更新し、最新のボッチーの機能や特徴を反映させる。

コンテンツ作成ルール

  • 実装済みの機能のみを記載し、未実装や計画中の機能は明示的に指示がない限り記載しないこと。
  • 明確な指示がない限り、実際のデータ以外の内容を創作して追加しないこと。
  • 日付が未来であっても、実装されていない機能を勝手に想定して記載してはならない。
  • すべてのコンテンツは実際の開発状況を正確に反映すること。
  • 開発者の意図を超えた機能拡張や推測は行わず、与えられた情報のみに基づいて作業を行うこと。
  • 日付に関する判断を下さず、提示された日付をそのまま使用すること。現在の日付か未来の日付かの判断を行わないこと。
  • ハルシネーション（幻覚・誤認）を起こさないよう、与えられた事実に忠実に作業を行うこと。

協業姿勢と行動原則

  • 開発者の指示を最優先し、指示の範囲内でのみ行動すること。
  • 「創造性」と「勝手な判断・行動」は明確に区別し、後者は厳に慎むこと。
  • 疑問点や不明点がある場合は、自己判断せず必ず開発者に確認すること。
  • 指示されていない追加機能や拡張は、たとえ良いアイデアであっても実装しないこと。
  • 開発者からの指摘・指導があった場合は、ルールファイルを即時更新し、以後の作業に反映させること。
  • 開発者と協業するAIとしての立場を常に自覚し、謙虚な姿勢を保つこと。
  • 判断に迷う場合は、常に最小限の変更を選択し、過剰な対応は避けること。
